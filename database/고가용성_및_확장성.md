# 고가용성(High Availabilty)과 확장성(Scalability)

## 1️⃣ 고가용성

<mark>**일부 오류로 인해 서비스가 중단을 막고 지속적인 운영이 가능한 성질**</mark></br>

고가용성을 위해 DB Server를 다중화하는 작업을 한다. 이를 통해 하나의 DB 서버가 죽더라도 서비스가 중단 되지 않게 한다. 

고가용성을 확보를 위한 DB 다중화 방식으로는 DB 클러스트링과 DB 리플리케이션이 있다. 

</br>

- **DB 클러스트링** : 여러 대의 DB 서버를 마치 한 대의 서버처럼 동작
  - DB 서버 수평적 확장에 중점
  - DB 서버의 FailOver 처리, 서버 간 Load Balancing을 주로 고려
  - 스토리지가 다중화되지 않아 데이터가 유실될 수 있는 문재가 존재
  
- **DB 리플리케이션** : DB서버와 저장소를 포함한 전체를 다중화
  - DB 스토리지 내 데이터 복제에 중점
  - DB 시스템 간 데이터 동기화, 데이터 일관성 및 신뢰성을 주로 고려
  - DB 서버와 저장소 세트를 여러 개로 다중화하여, 데이터 손실의 문제를 해결하려고 함


</br>

## 2️⃣ 확장성 

<mark>**애플리케이션의이나 시스템이 적응을 통해 더 큰 부하를 처리할 수 있음을 의미**</mark></br>

데이터베이스의 성능은 애플리케이션의 전체 성능에 큰 영향을 미치기 때문에 데이터베이스 확장성은 대규모 트래픽을 처리하고, 장애 발생 시에도 안정적으로 서비스를 제공하기 위해 필수적인 요소이다.

</br>
